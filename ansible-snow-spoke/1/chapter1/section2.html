Settting up ServiceNow

Setting up ServiceNow for integration with Ansible Automation Platform (AAP) via the Spoke involves configuring authentication. This setup ensures that ServiceNow can seamlessly trigger and interact with automated workflows in AAP, allowing for efficient incident management, provisioning, and end-to-end IT operations automation. Proper configuration of roles, OAuth settings, and connectivity is essential to establish a reliable and secure Spoke connection between the two platforms.

  1. Login to Service now using the credential.

    • Navigate to the ServiceNow Store and search for "Ansible Spoke" if its not installed and select the latest version and click Get to install it.

  2. Open the certificate file downloaded in the previous step, and copy the file contents carefully, ensuring there are no extra spaces or line breaks.

  3. Navigate to System Definition → Certificates in ServiceNow to display a list of all certificates used by the platform. Click the blue New button and enter the required details as shown in the following screenshot:

  4. Navigate to Connections & Credentials → Connection & Credential Aliases. Click the existing AnsibleTowerAlias alias. In the resulting dialog window, ensure the following fields are filled in:

    1. In the Related Links section, select Create New Connection & Credential and enter the following information:

      Parameter

      AAP 2.6

      Connection Name

      <provider-name> Spoke Connection

      Connection URL

      https://<AAP-URL>;

      Credential Name

      <provider-name> Spoke Credentials

      Application Registry Name

      <provider-name>

      OAuth Client ID

      The Client ID you got from AAP

      OAuth Client Secret

      The Client Secret you got from AAP

      OAuth Entity Profile Name

      Ansible Entity Profile

      OAuth Entity Scope

      write

      Authorization URL

      https://<AAP-URL>/o/authorize/

      Token URL

      https://<AAP-URL>/o/token/

      OAuth Redirect URL

      https://<snow_instance_id>.service-now.com/oauth_redirect.do

    2. Select Create and Get OAuth Token to complete the Ansible Spoke setup. A new window prompts you to authorize ServiceNow against your Ansible Automation Platform instance or cluster.

      Click the Authorize—this button that may initially fail with a 401 error, which is expected at this stage.

      1. Ensure that the jobs launch correctly in AAP 2.6 or later, update the API endpoint. Navigate to All → Connections & Credentials → Connections, open your newly created connection, and in the Attributes section, change the version from v2 to controller/v2. Click Update to save the changes.

      2. Go to System OAuth → Application Registry, and select the application you created earlier. Remove the value in the OAuth API Script field, and click Update.

      3. Navigate to All → Connections & Credentials → Credentials, and open your newly created credential. In the Related Links section, click Get OAuth Token. When prompted, click Authorize again—this time

        The process should complete successfully and generate the OAuth token.
