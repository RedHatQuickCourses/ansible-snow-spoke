Settting up ServiceNow

Setting up ServiceNow for integration with Ansible Automation Platform via the Spoke involves configuring authentication. This setup ensures that ServiceNow can seamlessly trigger and interact with automated workflows in AAP, allowing for efficient incident management, provisioning, and end-to-end IT operations automation. Proper configuration of roles, OAuth settings, and connectivity is essential to establish a reliable and secure Spoke connection between the two platforms.

  1. Login to Service now Using the credentias shared .

  2. Open the certificate file downloaded in the previous step, and copy its contents carefully, ensuring there are no extra spaces or line breaks.

  3. Navigate to System Definition → Certificates in ServiceNow. This will display a list of all certificates used by the platform. Click the blue New button and enter the required details as shown in the screenshot below:

  4. Navigate to Connections & Credentials-→Connection & Credential Aliases. Click the existing "AnsibleTowerAlias" alias. In the resulting dialog window, ensure the following fields are filled in:

    1. Under Related Links select "Create New Connection & Credential" and enter in the following information:

    2. Select Create and Get OAuth Token to complete the Ansible Spoke setup. A new window will appear prompting you to authorize ServiceNow against your Ansible Automation Platform instance or cluster. Click Authorize—this may initially fail with a 401 error, which is expected at this stage.

      1. To ensure jobs launch correctly in AAP 2.5 or later, update the API endpoint. Navigate to All → Connections & Credentials → Connections, open your newly created connection, and in the Attributes section, change the version from v2 to controller/v2. Click Update to save the changes.

      2. Next, go to System OAuth → Application Registry and select the application you created earlier. Remove the value in the OAuth API Script field, then click Update.

      3. Finally, navigate to All → Connections & Credentials → Credentials and open your newly created credential. Under Related Links, click Get OAuth Token. When prompted, click Authorize again—this time, the process should complete successfully and generate the OAuth token.

  5. Navigate to Service Catalog → Catalog Definitions → Maintain Items in ServiceNow. On the resulting list page, click the blue New button. In the dialog that appears, fill in the following fields:

  6. Navigate to Process Automation → Flow Designer in ServiceNow. This will open in a new tab. Click Actions, search for “Launch Job Template”, and select the corresponding action.

    1. Once the action loads, click the three dots in the top-right corner and select Copy Action. Provide a new name and choose Ansible Spoke as the application in the dialog box.

    2. Enter in a name and select Ansible Spoke as the Application in the box that opens.

    3. Select Launch Workflow on the left-hand side

      1. In the copied action, select Launch Workflow from the left panel. Scroll down to the Resource Path section and update the URL so it appears as: api/controller/v2/job_templates/step→Pre-Processing→workflow_job_template_id/launch

  7. Return to the Flow Designer home by clicking Home (top-left). Click the blue New button and select Flow.

  8. In the new flow, click Add a Trigger, choose Service Catalog, then enable Advanced Options and select Run flow in background (default).

  9. Under Actions, click Add an Action, Flow Logic, or Subflow, select Action, and choose Launch Job Template from the Ansible Spoke options. Click the blue Done button.

  10. Click on the blue Done Button

  11. Navigate back to Service Catalog → Catalog Definitions → Maintain Items, and open the catalog item you created earlier. Go to Process Engine, set the Flow field to the flow you just created, then right-click the header area and click Save.

  12. To test the setup, go to Self-Service → Service Catalog, search for your catalog item, and click Order Now. You should see the request submitted in ServiceNow, and the corresponding job execution triggered in Ansible Automation Platform.

Congratulations! After completing these steps, you can successfully use a ServiceNow Catalog Item to launch job templates in Ansible Automation Platform via the Ansible Spoke. This integration enables end users to leverage a familiar interface while triggering powerful backend automation, significantly improving efficiency and reducing time to value across the organization.
