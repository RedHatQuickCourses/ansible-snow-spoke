Operational Flow: From Trigger to Execution

The flow doesn’t just live inside ServiceNow. It can reach out via REST API to the Ansible Automation Platform (AAP) to launch playbooks, effectively bridging the gap between an ITSM request and infrastructure-level execution. This orchestration ensures that business logic is strictly enforced while providing a transparent, auditable trail of every automated step taken across the enterprise.

  1. Navigate to Process Automation → Flow Designer in ServiceNow, which opens a new tab.

    In the newly opened tab, click Actions, search for “Job" , and select the corresponding action:

    1. Once the action loads, click the three dots in the top-right corner and select Copy Action. Provide a new name and choose Ansible Spoke as the application in the dialog box and click the Copy button.

    2. Change the Job Template ID to String and click on save.

  2. Return to the Flow Designer home by clicking Home (top-left). Click the blue New button, select Flow, and add the details according to the following screenshots.

  3. In the new flow, click Add a Trigger, choose Service Catalog, then enable Advanced Options.

    By default, the Run flow in background (default) is selected.

  4. Under Actions, click Add an Action, Flow Logic, or Subflow, select Action, and choose Launch Job Template from the Ansible Spoke options. Click the blue Done button.

  5. Click on the blue Done Button

  6. Navigate back to Service Catalog → Catalog Definitions → Maintain Items, and open the catalog item you created earlier. Go to Process Engine, set the Flow field to the flow that you created, then right-click the header area, and click Save.

  7. Go to Self-Service → Service Catalog, Add a new catalog and then click on the catalog item you just created and click on Order Now. You should see the request submitted in ServiceNow, and the corresponding job execution triggered in Ansible Automation Platform.

Congratulations! After completing these steps, you can successfully use a ServiceNow Catalog Item to launch job templates in Ansible Automation Platform via the Ansible Spoke. This integration enables end users to leverage a familiar interface while triggering powerful backend automation, significantly improving efficiency and reducing time to value across the organization.
